# 界面与使用流程

## 1. 总体流程

```text
打开软件
  → 从模板添加供应商实例
  → 进入上游模型同步或手动添加模型
  → 创建统一模型并绑定候选
  → 按需配置辅助模型工作流
  → 创建 API Token
  → 在路由状态验证协议转换
  → 配置 Claude Code 或其他客户端
```

## 2. 页面定义

### 2.1 仪表盘

保留原有摘要能力，并按新对象改名。展示运行地址、服务状态、供应商实例数、可用上游模型、统一模型、辅助模型、近 24 小时调用、成功率、成本、预算告警和最近错误。

### 2.2 供应商

两个区域：

1. **模板目录**：搜索、协议/地区/能力筛选、验证状态、默认地址、认证方式和“添加”操作。
2. **供应商实例**：名称、模板、协议、Base URL、凭据状态、启用状态、连接状态、编辑、测试、复制和删除。

添加模板后弹出实例表单；API Key 只允许写入，不在详情中明文返回。手动供应商必须选择 OpenAI-Compatible、Anthropic Messages、Gemini 或自定义协议，并允许自定义请求头。

### 2.3 上游模型

顶部先选择供应商实例。主要操作：测试连接、拉取模型、手动新增、批量启停、删除和刷新。

列表展示模型 ID、显示名、能力、上下文、价格、启用状态、远端状态、引用数量和同步时间。被统一模型或辅助模型引用的记录不可直接删除。

### 2.4 统一模型

统一模型表单包含名称、描述、能力、入口协议、多协议开关、路由模式、Combo 策略、偏好档位、会话粘性和启用状态。

候选绑定使用上游模型选择器，不再手填 Provider ID 和上游模型字符串。每个候选可设置优先级、权重、能力覆盖和启用状态。

### 2.5 辅助模型

顶部设置模式：关闭、按统一模型、全局模型池。默认全局模型池。

配置顺序：选择供应商 → 选择上游模型 → 选择辅助能力 → 选择工作流 → 设置优先级和顺序。页面显示哪些统一模型会使用该配置，并提供模拟匹配结果。

### 2.6 API Token

创建 Token、一次性复制明文、配置 scopes/有效期/预算归属、启停和删除。页面明确区分“客户端 Token”和“供应商 API Key”。

### 2.7 路由状态

包含四个视图：

- 统一模型与入口协议矩阵。
- 候选上游模型和 Combo 计算结果。
- 健康、熔断、额度和失败切换状态。
- 协议转换测试台。

测试台展示入口原文、内部规范对象、上游请求、上游响应和最终响应；测试模式默认不调用真实供应商，可使用 Mock。

### 2.8 调用日志

按时间、状态、统一模型、上游模型、供应商、协议和 Token 筛选。详情展示路由决策、重试链、转换摘要、基础辅助链、错误代码和失败阶段。默认不保存完整敏感 Prompt/Response。

### 2.9 价格与用量

维护模型价格并展示按供应商、上游模型、统一模型、协议和 API Token 聚合的请求、Token 与成本。允许手工价格覆盖，保留价格来源和生效时间。

### 2.10 预算控制

支持全局、API Token、统一模型和供应商实例范围。动作包括拒绝、警告、回退免费候选和回退低价候选；任何回退仍必须满足能力和协议要求。

### 2.11 Agent 配置

Claude Code 区域包含主模型、Opus、Sonnet、Haiku 四个统一模型选择器，Base URL 默认使用当前运行地址。提供预览、写入、备份和恢复。

Codex、Gemini CLI、OpenCode 显示扩展卡片和统一接口占位，但不得假装已完成写入功能。

### 2.12 系统设置

分组：

- 运行：首选端口、当前端口、数据目录和上传限制。
- 桌面：托盘、开机自启动、单实例状态。
- 安全：主密钥状态、数据库备份、恢复。
- WebDAV：服务器、用户名、加密密码、备份密码、测试、同步和恢复。
- 系统：版本、构建信息、诊断导出。

## 3. 关键交互规则

- 所有保存操作显示明确成功或字段级错误。
- 删除被引用对象时返回引用列表，不能静默级联。
- 拉取、测试、同步和备份显示进行状态，不允许重复提交。
- 协议或能力不匹配时提供错误原因和手动处理建议，不自动改选统一模型。
- 计划中、兼容模式和已验证供应商使用不同状态标签。
- 页面只使用中文术语，不保留旧菜单入口。
