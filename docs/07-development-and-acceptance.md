# 开发计划与验收

## 1. 交付方式

这是一次完整重构交付。内部可以按阶段实现和验证，但不得把阶段结束当作产品交付；除明确延期项外，所有阶段完成并通过总验收后才算完成。

不得把真实供应商调用作为当前验收前置条件。使用 Mock、HTTP 模拟服务器和固定协议样例验证；不得使用聊天中出现过的旧测试密钥。

## 2. 实施顺序

### 阶段 A：基线与数据库

- 保存当前测试基线和构建结果。
- 新建 schema generation 和旧库备份/重建流程。
- 建立供应商实例、上游模型、统一模型候选、辅助模型和系统设置新表。
- 删除新业务路径对 Connection/Node 的依赖。

验收：重建成功、失败回滚、旧库备份存在、默认设置正确、引用约束测试通过。

### 阶段 B：供应商与上游模型

- 建立版本化模板目录。
- 实现模板筛选、手动供应商和多个同模板实例。
- 实现加密凭据、自定义头、连接测试。
- 实现模型拉取、手工添加、幂等同步、远端消失标记和批量操作。
- 重写供应商和上游模型页面。

验收：相同模板多 Key 互不覆盖；无模型接口时可手动维护；被引用模型不可删除。

### 阶段 C：统一模型与路由

- 候选直接引用 `upstream_model_id`。
- 实现多入口协议配置。
- 迁移和统一所有 Combo 策略。
- 健康、熔断、额度、预算、会话粘性绑定新模型结构。
- 重写统一模型和路由状态页面。

验收：跨供应商候选、各策略排序、失败切换、能力过滤和解释结果测试通过。

### 阶段 D：协议内核

- 定义 Canonical Request/Response/Event。
- 所有网关端点接入统一执行管线。
- 建立协议转换器、能力矩阵和结构化错误。
- 实现 dry-run 协议转换测试台。

验收：Chat、Responses、Messages、Gemini、Embeddings、Files、Images、Audio、Moderations、Rerank、Search、Batches、WebSocket、Video、Music 均通过契约测试；不兼容场景明确失败。

### 阶段 E：辅助模型

- 实现 disabled、per-unified-model、global-pool 三模式。
- 实现供应商 → 上游模型 → 辅助配置页面。
- 实现可排序预设工作流和能力规划。
- 支持中间处理和终端能力步骤。
- 实现失败阶段错误和基础日志。

验收：每种工作流有成功、缺配置、超时、上游失败和不兼容测试；不得调用未配置模型。

### 阶段 F：Token、日志、价格、预算

- API Token 页面移动到辅助模型之后。
- 日志和聚合查询适配新引用。
- 价格、用量和预算作用于供应商实例、上游模型和统一模型。
- 保留基础辅助链日志，不做延期的分步成本归集。

验收：鉴权、过期、scope、成本聚合和预算动作测试通过。

### 阶段 G：Agent 与系统设置

- Claude Code 主模型/Opus/Sonnet/Haiku 映射。
- 预览、备份、原子写入和恢复。
- Codex、Gemini CLI、OpenCode 扩展入口。
- 系统设置合并 WebDAV、运行、桌面、安全和诊断。
- WebDAV 全量加密备份与安全恢复。

验收：端口变化自动更新并备份 Agent 配置；WebDAV 错密码、损坏和冲突不破坏本地数据。

### 阶段 H：桌面与发布

- 固定优先 8080、冲突自动换端口。
- 单实例和 IPC 唤醒。
- 托盘、自启动、后台启动和优雅退出。
- 单文件打包和敏感信息扫描。

验收：重复启动只保留一个后端；无 Python/Node 环境能运行；数据只写入 `%USERPROFILE%\.apiswitch`。

### 阶段 I：全量回归与文档

- 更新 API 契约和实际实现状态。
- 运行后端、前端、构建和打包验证。
- 检查文档链接、数据库初始化和示例。
- 输出最终 EXE 路径、大小、SHA-256 和已知限制。

## 3. 质量门槛

每阶段都必须：

- 新增或更新自动化测试。
- 不破坏已完成范围。
- 运行相关 pytest。
- 运行前端测试和 TypeScript 检查。
- 涉及前端时运行生产构建。
- 涉及桌面时运行真实 EXE 冒烟测试。
- 执行 `git diff --check`。

最终至少执行：

```powershell
python -m pytest backend\tests -q
cd frontend
npm run test
npm run build
cd ..
.\scripts\package-desktop.ps1 -Clean
```

## 4. 测试矩阵

- 数据库：首次创建、旧库备份重建、失败回滚、引用约束。
- 安全：密钥加密、Token 哈希、脱敏、SSRF/头注入、路径穿越。
- 供应商：模板、手工实例、重复模板、测试、模型同步。
- 路由：所有 Combo、熔断、预算、额度、会话、候选失败。
- 协议：非流式、SSE、WebSocket、工具、多模态、文件和错误转换。
- 辅助模型：三模式、排序、能力匹配、每个工作流和失败阶段。
- WebDAV：加密往返、错误密码、损坏、版本、冲突和恢复回滚。
- Agent：四模型映射、路径、预览、备份、恢复、动态端口。
- 桌面：单实例、托盘、自启动、后台模式、端口占用、退出。
- UI：12 个菜单、核心表单、加载/错误/空状态和响应式布局。

## 5. 完成定义

只有同时满足以下条件才可报告完成：

1. 十二个页面均使用新业务结构且无旧入口。
2. 所有确认协议进入同一管线。
3. 三种辅助模型模式可执行。
4. 数据库按决策备份并全量重建。
5. WebDAV 全量加密备份可恢复。
6. Claude Code 配置能自动跟随端口。
7. 单实例、托盘、自启动和单文件发布通过实际验证。
8. 所有自动化测试和生产构建通过。
9. 文档与代码一致。

## 6. 唯一延期项

辅助调用链每一步的独立 Token、成本、延迟和预算归集。不延期基础链路日志、失败阶段和总请求现有统计。
