# 桌面运行、安全与备份

## 1. Windows 桌面生命周期

最终产物是单文件 `dist\APISwitch.exe`，包含后端、前端、Python 运行时、协议模块、托盘依赖和静态资源。

运行数据全部位于：

```text
%USERPROFILE%\.apiswitch
├─ apiswitch.db
├─ master.key
├─ runtime.json
├─ files\
├─ backups\
├─ logs\
└─ temp\
```

禁止把用户数据写入 EXE 所在目录或 PyInstaller 临时解包目录。

## 2. 单实例

- 使用 Windows 当前用户范围的命名互斥体。
- 首实例创建本地 IPC 端点。
- 重复启动向首实例发送 `show`，首实例恢复并聚焦窗口，随后重复进程退出。
- 锁失效或进程崩溃后允许新实例安全接管。

## 3. 端口管理

1. 首选 `127.0.0.1:8080`。
2. 已占用时探测可用回环端口。
3. 启动成功后写入 `runtime.json`。
4. 前端通过同源调用，不编译固定端口。
5. Agent 配置使用 `runtime.json` 中的当前 Base URL。
6. Base URL 变化时备份并更新所有已启用 Agent 配置。

管理界面必须显示当前地址并提供复制按钮。

## 4. 托盘与自启动

- 最小化和窗口关闭操作默认隐藏到托盘。
- 托盘左键恢复；右键提供打开、隐藏、复制网关地址、开机自启动和退出。
- 完全退出时停止调度器、等待正在写入的数据库事务、停止后端并移除托盘。
- 自启动写入当前用户 Run 注册项，命令使用 `--background`。
- 移动 EXE 后，重新启用自启动会更新路径。

## 5. 本地密钥

- 首次启动生成随机主密钥，不编译固定密钥。
- 主密钥文件只供当前用户访问；实现应优先结合 Windows DPAPI 或严格 ACL。
- 供应商 API Key、OAuth、代理凭据、自定义敏感头和 WebDAV 密码使用认证加密。
- 解密失败必须报错，不能覆盖或重新生成导致旧密文永久不可读。
- 诊断包不得包含主密钥、数据库、凭据或完整请求内容。

## 6. API 安全

- Admin API 默认只允许 loopback。
- Gateway API 始终要求具有 `gateway:invoke` scope 的 Bearer Token。
- Token 只保存强哈希，明文只返回一次。
- 自定义 Base URL 和请求头必须防 SSRF、CRLF 注入和敏感头泄漏。
- 错误、日志和测试输出统一脱敏。

## 7. 本地备份

- 数据库备份使用 SQLite Backup API，不直接复制正在写入的数据库文件。
- 备份文件包含版本、时间、schema generation 和 SHA-256 清单。
- 恢复前自动创建当前状态备份。
- 恢复在临时目录验证，完成后原子替换。

## 8. WebDAV 全量加密归档

归档范围：数据库、主密钥、文件、日志和必要设置。流程：

```text
一致性快照
→ 文件清单与校验值
→ 压缩
→ 使用独立备份密码派生密钥
→ AEAD 加密
→ WebDAV 上传
```

要求：

- 使用带随机 salt、合理成本的 KDF。
- 每次归档使用随机 nonce。
- 元数据仅暴露版本、时间和非敏感校验信息。
- 支持远端列表、差异预览、上传、下载、冲突策略和同步日志。
- 新设备恢复必须输入备份密码。
- 密码错误、校验失败或版本不兼容时不得修改本地数据。

## 9. 数据库全量重建

检测旧 generation 时先备份旧数据库，再创建新库，不迁移业务数据。UI 首次启动显示重建说明和旧备份路径。重建失败应阻止正常运行并提供恢复指引。

## 10. 发布检查

- 构建前清除 `VITE_API_BASE_URL` 和 `VITE_ADMIN_TOKEN` 等开发机变量。
- 构建产物中扫描已知测试密钥和常见密钥前缀。
- 验证单文件在无源码、无 Python、无 Node 环境中启动。
- 验证 UI、健康检查、托盘、单实例、动态端口和 `%USERPROFILE%\.apiswitch`。
- 生成 SHA-256，并记录构建版本和时间。
